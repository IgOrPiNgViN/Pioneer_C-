## ВАРИАНТ 1

### Теоретическая часть

**Вопрос 1** (Урок 1: Введение)  
- Компиляция — процесс преобразования исходного кода в машинный код
- В результате создаётся исполняемый файл (.exe в Windows)
- `#include` — директива препроцессора, которая включает содержимое другого файла (обычно заголовочного) в программу

**Вопрос 2** (Урок 2: Переменные)  
- `float` — 4 байта, точность ~7 знаков после запятой
- `double` — 8 байт, точность ~15 знаков после запятой
- `double` точнее, чем `float`
- Запись: `3.14`, `2.5f` (для float), `1.0e-5` (научная нотация)

**Вопрос 3** (Урок 3: Операторы)  
- Составные операторы объединяют арифметическую операцию с присваиванием
- `+=`: `x += 5;` эквивалентно `x = x + 5;`
- `-=`: `x -= 3;` эквивалентно `x = x - 3;`
- `*=`: `x *= 2;` эквивалентно `x = x * 2;`

**Вопрос 4** (Урок 6: Массивы)  
- При выходе за границы — неопределённое поведение (может быть мусор, ошибка, сбой)
- Размер массива: `sizeof(arr) / sizeof(arr[0])` = `40 / 4 = 10`

**Вопрос 5** (Урок 10: Функции)  
- Область видимости — часть программы, где переменная доступна
- Локальная — объявлена внутри функции, видна только в ней, уничтожается при выходе
- Глобальная — объявлена вне функций, видна везде, существует всё время работы программы

**Задание 2:** `51`

## ВАРИАНТ 2

### Теоретическая часть

**Вопрос 1** (Урок 2: Переменные)  
- Приведение типов — преобразование значения из одного типа в другой
- Явное приведение: `int x = (int)3.7;` или `int x = static_cast<int>(3.7);`
- Дробная часть отбрасывается (не округляется): 3.7 → 3

**Вопрос 2** (Урок 3: Операторы)  
- Приоритет: `*`, `/`, `%` выполняются раньше `+`, `-`
- `2 + 3 * 4 - 8 / 2` = `2 + 12 - 4` = `10`

**Вопрос 3** (Урок 4: Условия)  
- Тернарный оператор: `условие ? значение_если_true : значение_если_false`
- Переписанный код: `y = (x > 0) ? 1 : -1;`

**Вопрос 4** (Урок 7: Строки)  
- Конкатенация: `string s = s1 + s2;` или `s1 += s2;`
- Добавление символа: `s.push_back('a');` или `s += 'a';`

**Вопрос 5** (Урок 8: Указатели)  
- `nullptr` — нулевой указатель, не указывает ни на что
- Нужен для инициализации указателя, который пока не имеет адреса
- Разыменование nullptr вызовет ошибку (crash, segmentation fault)

## ВАРИАНТ 3

### Теоретическая часть

**Вопрос 1** (Урок 1: Введение)  
- Комментарии — текст, игнорируемый компилятором, для пояснения кода
- Однострочный: `// это комментарий`
- Многострочный: `/* это многострочный комментарий */`

**Вопрос 2** (Урок 3: Операторы)  
- Короткое замыкание — прекращение вычисления, когда результат уже известен
- `&&`: если первый операнд false, второй не вычисляется (результат уже false)
- `||`: если первый операнд true, второй не вычисляется (результат уже true)

**Вопрос 3** (Урок 5: Циклы)  
- `while` — проверяет условие ДО выполнения тела
- `do-while` — проверяет условие ПОСЛЕ выполнения тела
- Тело `do-while` выполнится минимум один раз, даже если условие изначально ложно

**Вопрос 4** (Урок 8: Указатели и ссылки)  
- Указатель хранит адрес, может быть переназначен и быть null
- Ссылка — псевдоним переменной, не может быть null, нельзя переназначить
- После инициализации ссылка всегда указывает на тот же объект

**Вопрос 5** (Урок 9: Структуры)  
- Да, можно создавать массивы структур: `Student students[10];`
- Доступ к полю: `students[2].name` (индексация с 0, третий элемент — индекс 2)

## ВАРИАНТ 4

### Теоретическая часть

**Вопрос 1** (Урок 2: Переменные)  
- `char` — тип для хранения одного символа, занимает 1 байт
- Символ: `char c = 'A';`
- ASCII-код: `int code = (int)'A';` → 65
- Или: `char c = 65;` → символ 'A'

**Вопрос 2** (Урок 4: Условия)  
- Вложенный if — условие внутри другого условия
```cpp
if (x > 0) {
    // положительное
} else if (x < 0) {
    // отрицательное
} else {
    // ноль
}
```

**Вопрос 3** (Урок 6: Массивы)  
- Инициализация: `int arr[2][3] = {{1, 2, 3}, {4, 5, 6}};`
- Доступ: `arr[1][2]` (вторая строка — индекс 1, третий столбец — индекс 2)

**Вопрос 4** (Урок 7: Строки)  
- Метод `find()`: `size_t pos = str.find("подстрока");`
- Если не найдено, возвращает `string::npos`

**Вопрос 5** (Урок 10: Функции)  
- Перегрузка — несколько функций с одинаковым именем, но разными параметрами
- Можно, если отличаются: количество параметров, типы параметров, порядок типов
- Нельзя различать только по типу возвращаемого значения

**Задание 2:** `12`

Положительные: 4, 7, 3, 6 → positive = 4 + 7 + 3 + 6 = 20

**Задание 3:** `48`
```
a = 8, b = 3

swap(a, b): a и b меняются местами
a = 3, b = 8

increase(a, b): a передаётся по ссылке, b по значению
a = 3 + 8 = 11 → a = 11
y = 8 * 2 = 16 (но b не меняется, т.к. по значению)
b остаётся 8

a * b = 11 * 8 = 88

Исправление:
После swap: a = 3, b = 8
increase(a, b): x = &a, y = b (копия)
x = 3 + 8 = 11 → a = 11
y внутри функции = 8 * 2 = 16, но b вне функции не меняется

a * b = 11 * 8 = 88
```
